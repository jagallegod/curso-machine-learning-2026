Árboles de Regresión en R


1. Introducción


  • Los métodos de regresión son la herramienta más popular para predicción numérica

  • Pero pueden tener problemas

  • En particular cuando se tienen muchos predictores

  • O cuando existen relaciones complejas (no-lineales) entre outcomes y predictores

  • Los árboles de regresión y los modelos de árboles son una alternativa interesante

  • Al igual que en clasificación, la esencia está en dividir las observaciones

  • De tal forma que pasemos de un grupo grande heterogéneo, a grupos pequeños homogéneos

  • La diferencia está en la regla de predicción al llegar a los nodos términales

  • En los árboles de regresión, se calcula un promedio

  • En los modelos de árbol, se corre una regresión en cada nodo terminal

  • Veremos cómo implementar los árboles de regresión en R

  • Para ello, usaremos datos de vinos. El objetivo es predecir la calidad de vinos blancos

  • La idea es crear un sistema que simule lo que hacen los catadores de vino

  • De tal forma que con base en ciertas características de los vinos se prediga su calidad


2. Datos


  • Usaremos la base whitewines.csv, del UCI Machine Learning Data Repository

  • La base contiene información de diferentes vinos portugueses, incluyendo la calificación de calidad hecha por expertos

  • Se incluyen 11 propiedades químicas para 4898 muestras de vino

  • Cada muestra fue calificada por un panel de mínimo tres jueces que cataron ciegamente cada uno y calificaron en una escala de 0 a 10

  • Carguemos los datos. Como siempre, no olviden fijar el directorio de trabajo

3. Árbol de Regresión


  • Comencemos entrenando un árbol de regresión

  • Para ello usaremos el paquete rpart

  • Instalémoslo con install.packages("rpart") y carguémoslo


library(rpart)


  • El modelo se construye en dos etapas

  • En la primera etapa construimos el árbol usando los datos de prueba

  • Para ello usamos la función rpart que tiene la siguiente sintaxis: m <- rpart(dv ~ iv, data=mydata)

  • Con esto obtendremos un árbol de regresión que puede usarse para hacer las predicciones

  • Estas las hacemos con la función predict, según la siguiente sintaxis: p <- predict(m, test, type="vector")

  • Que en este caso devuelve el vector de predicciones númericas para cada ejemplo en el conjunto de prueba

  • Pasemos ahora sí a entrenar nuestro modelo


m.rpart <- rpart(quality ~ ., data=wine_train)

  • Ahora, usando los datos de prueba, podemos evaluar el desempeño del modelo

  • Usamos la función predict() para ello


p.rpart <- predict(m.rpart, wine_test)


  • Se ha creado un vector con la calidad pronosticada para cada muestra de vino en el conjunto de prueba


summary(p.rpart)

##   5.349   5.349   5.717   5.868   6.233   6.602


  • Se ve que hay poca varianza, lo cual es problemático

  • Va a ser imposible pronosticar cuáles vinos son muy buenos o muy malos, que los hay en la base de prueba


summary(wine_test$quality) 
##    Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
##   3.000   5.000   6.000   5.879   6.000   8.000


  • Una primera forma de ver si nuestro modelo lo hace bien es examinando la correlación entre pronósticos y valores reales


cor(p.rpart, wine_test$quality)
## [1] 0.5084365


  • Es una correlación alta pero no como para sacar el carro de bomberos

  • Necesitamos una medida más precisa del desempeño del modelo

  • Usaremos el error absoluto medio (mean absolute error MAE):

  • \(MAE=\frac{1}{n}\sum_{i=1}^n|e_i|\)

  • donde \(e_i\) es el error de predicción para la observación \(i\)

  • Creemos una función para calcular el MAE


MAE <- function(actual, predicted){
  mean(abs(actual-predicted))
}


  • Implementemos la función para nuestro modelo:


MAE(p.rpart, wine_test$quality)
## [1] 0.6080502


  • Obtenemos un MAE de 0.62. En promedio, la diferencia entre nuestras predicciones y las calidades reales de los vinos de prueba es 0.62

  • Recordemos que la escala va de 0 a 10, así que no lo estamos haciendo tan mal

  • Pero tampoco es tan impresionante en comparación con predecir la media del conjunto de prueba


mean(wine_train$quality)
## [1] 5.8776
MAE(5.8912, wine_test$quality)
## [1] 0.6665993


  • Dicho modelo ingenuo logra un MAE de 0.69. No es muy distinto al que alcanzamos nosotros
