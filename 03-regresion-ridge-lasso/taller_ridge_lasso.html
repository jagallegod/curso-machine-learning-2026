Ridge y Lasso en R

1. Introducción


  • Ridge y Lasso son extensiones del modelo de regresión lineal que atacan los problemas de la alta dimensionalidad (muchas variables frente al número de observaciones): alta varianza en las predicciones y dificultad de interpretación.

  • Ambos introducen una penalidad sobre el tamaño de los coeficientes, halándolos hacia 0.

  • Bajo Ridge los coeficientes tienden a 0 a medida que pesa más la penalidad, pero nunca son exactamente 0.

  • Bajo Lasso algunos coeficientes sí se vuelven exactamente 0.

  • La clave es la elección del parámetro \(\lambda\), que mide el peso de la penalidad.

hitters <- na.omit(hitters)

  • La sintaxis de glmnet es distinta: hay que construir la matriz de atributos con model.matrix() (que además crea las dummies de las categóricas) y el vector de outcomes:


x <- model.matrix(Salary ~ ., hitters)[, -1]
y <- hitters$Salary


3. Regresión Ridge


  • En glmnet(), el parámetro alpha define el modelo: alpha = 0 es Ridge y alpha = 1 es Lasso.

  • También definimos una grilla de valores de \(\lambda\), desde valores enormes (modelo casi sin predictores) hasta cercanos a 0 (OLS). Creamos 100 valores:


grid <- 10^seq(10, -2, length = 100)


  • Estimamos los 100 modelos Ridge (uno por cada \(\lambda\)):


hitters_ridge <- glmnet(x, y, alpha = 0, lambda = grid)


  • Cada \(\lambda\) genera coeficientes distintos: a mayor \(\lambda\), más pequeños. Comparemos el \(\lambda\) en la posición 50 con el de la 60 (recuerda que, por cómo armamos la grilla, índices mayores corresponden a \(\lambda\) menores):


hitters_ridge$lambda[50]
## [1] 11497.57
hitters_ridge$lambda[60]
## [1] 705.4802

  • Para entrenar y probar, dividimos la base por la mitad (entrenamiento / prueba), fijando la semilla:


set.seed(123)
train <- sample(1:nrow(x), nrow(x) / 2)
test <- (-train)
y_test <- y[test]


  • Reestimamos Ridge solo con el entrenamiento y graficamos cómo decrecen los coeficientes al crecer \(\lambda\):


hitters_ridge <- glmnet(x[train, ], y[train], alpha = 0, lambda = grid)
plot(hitters_ridge, xvar = "lambda")


  • ¿Cuál es el \(\lambda\) óptimo? Lo elegimos por validación cruzada con cv.glmnet():


set.seed(123)
cv_ridge <- cv.glmnet(x[train, ], y[train], alpha = 0)
bestlam <- cv_ridge$lambda.min
bestlam
## [1] 27.68026


  • Ese es el \(\lambda\) que minimiza el error de validación cruzada. El modelo asociado es:


predict(cv_ridge, s = bestlam, type = "coefficients")[1:20, ]

4. Lasso


  • El proceso es equivalente, pero con alpha = 1. Estimamos sobre el entrenamiento y graficamos:


hitters_lasso <- glmnet(x[train, ], y[train], alpha = 1, lambda = grid)
plot(hitters_lasso, xvar = "lambda")


  • A medida que \(\lambda\) crece, los coeficientes caen y algunos se vuelven exactamente 0. Busquemos el \(\lambda\) óptimo por validación cruzada:


set.seed(123)
cv_lasso <- cv.glmnet(x[train, ], y[train], alpha = 1)
bestlam <- cv_lasso$lambda.min
bestlam
## [1] 0.7883439


  • Y veamos el modelo asociado:


predict(cv_lasso, s = bestlam, type = "coefficients")[1:20, ]
